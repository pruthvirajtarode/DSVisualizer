Queue Data Structure

Queue Information

A queue is a fundamental linear data structure that operates based on the First-In-First-Out (FIFO) principle. This means the first element added to the queue is always the first one to be removed. Elements are inserted at the back, known as the 'rear' or 'tail', and removed from the front, called the 'head'. Queues are essential in scenarios where order matters, such as task scheduling, printer queues, and message processing systems.

Types of Queue

Simple Queue

A basic queue where elements are added at the rear and removed from the front. It follows strict FIFO principle and has a fixed size.

Circular Queue

A queue where the last element is connected to the first element, forming a circle. This helps in efficient memory utilization and prevents overflow.

Priority Queue

A special queue where elements are served based on their priority rather than their order of arrival. Higher priority elements are served first.

Double Ended Queue (Deque)

A queue that allows insertion and deletion from both ends. It combines the features of both stack and queue data structures.

Key Characteristics of Queues

  • FIFO Principle: First element added is the first to be removed
  • Two Main Operations: Enqueue (add) and Dequeue (remove)
  • Fixed Size: Can be implemented with a fixed or dynamic size
  • Efficient Operations: Both enqueue and dequeue operations are O(1)
  • No Random Access: Can only access elements at the front
  • Common Applications: Task scheduling, printer queues, message processing

Time Complexity

  • Enqueue: O(1) - Constant time operation
  • Dequeue: O(1) - Constant time operation
  • Front: O(1) - Constant time operation
  • Search: O(n) - Must traverse through elements
  • IsEmpty: O(1) - Constant time operation
  • Size: O(1) - Constant time operation

Applications of Queues

  • Task Scheduling in Operating Systems
  • Printer Queue Management
  • Message Processing Systems
  • Breadth-First Search in Graphs
  • Call Center Systems
  • Network Packet Processing

Queue Operations Pseudocode

Simple Queue

class SimpleQueue:
    def __init__(self, size):
        self.size = size
        self.queue = [None] * size
        self.front = self.rear = -1

    def enqueue(self, item):
        if self.is_full():
            return "Queue is full"
        if self.is_empty():
            self.front = self.rear = 0
        else:
            self.rear = (self.rear + 1) % self.size
        self.queue[self.rear] = item

    def dequeue(self):
        if self.is_empty():
            return "Queue is empty"
        item = self.queue[self.front]
        if self.front == self.rear:
            self.front = self.rear = -1
        else:
            self.front = (self.front + 1) % self.size
        return item

    def is_empty(self):
        return self.front == -1

    def is_full(self):
        return (self.rear + 1) % self.size == self.front

Priority Queue

class PriorityQueue:
    def __init__(self):
        self.queue = []

    def enqueue(self, item, priority):
        self.queue.append((item, priority))
        self.queue.sort(key=lambda x: x[1])

    def dequeue(self):
        if self.is_empty():
            return "Queue is empty"
        return self.queue.pop(0)[0]

    def peek(self):
        if self.is_empty():
            return "Queue is empty"
        return self.queue[0][0]

    def is_empty(self):
        return len(self.queue) == 0

    def update_priority(self, item, new_priority):
        for i, (x, p) in enumerate(self.queue):
            if x == item:
                self.queue[i] = (item, new_priority)
                self.queue.sort(key=lambda x: x[1])
                return True
        return False

Double Ended Queue (Deque)

class Deque:
    def __init__(self):
        self.items = []

    def add_front(self, item):
        self.items.insert(0, item)

    def add_rear(self, item):
        self.items.append(item)

    def remove_front(self):
        if self.is_empty():
            return "Deque is empty"
        return self.items.pop(0)

    def remove_rear(self):
        if self.is_empty():
            return "Deque is empty"
        return self.items.pop()

    def peek_front(self):
        if self.is_empty():
            return "Deque is empty"
        return self.items[0]

    def peek_rear(self):
        if self.is_empty():
            return "Deque is empty"
        return self.items[-1]

    def is_empty(self):
        return len(self.items) == 0

    def size(self):
        return len(self.items)
